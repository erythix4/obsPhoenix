Phoenix
LLM Observability Lab

Plateforme d'observabilité open-source pour applications LLM et RAG. Tracez, évaluez et déboguez vos pipelines IA en production - self-hosted, zéro envoi de données.

Pourquoi Phoenix ?

Les LLMs sont des boîtes noires. Quand votre RAG hallucine, quand vos réponses se dégradent en production, quand vous migrez de GPT-4 à Claude 3.5 - sans observabilité, vous volez à l'aveugle. Phoenix comble ce vide avec une approche 100 % open-source.

[ LLM ]
Tracing LLM
Capture chaque appel LLM, embedding, retrieval avec OpenTelemetry
[ EVAL ]
Évaluations
Score automatique : hallucination, relevance, toxicité, faithfulness
[ DATA ]
Datasets
Construire des jeux de données depuis les traces pour fine-tuning
[ A/B ]
Expériences
A/B testing de prompts, modèles, stratégies RAG
[ OSS ]
100% Open Source
Self-hosted, aucun envoi de données, souveraineté complète
[ API ]
Intégrations
OpenAI, Anthropic, LangChain, LlamaIndex, DSPy, CrewAI...
>
Positionnement dans l'écosystème
Phoenix se positionne entre LangSmith (propriétaire, OpenAI-centric) et Langfuse (évaluations moins riches). Son point fort : les évaluations LLM-as-a-judge intégrées et les expériences A/B sur datasets.

Cas d'usage typiques

RAG Pipeline
Tracer retrieval + generation, mesurer faithfulness et relevance
Chatbots
Détecter drift de qualité, hallucinations, toxicité en continu
Agents
Visualiser les tool calls, planification multi-étapes, boucles
Fine-tuning
Exporter des traces annotées comme dataset d'entraînement JSONL

Architecture Phoenix

Phoenix suit le standard OpenTelemetry (OTel) enrichi avec OpenInference - une convention sémantique spécifique aux LLMs développée par Arize AI. Tout instrument compatible OTel peut envoyer des traces à Phoenix.

Flux de données

Votre App\n(Python)
→
OpenInference\nInstrumentors
→
OTel Collector\n(gRPC/HTTP)
→
Phoenix\nServer UI

Hiérarchie Trace → Span

Trace: "Réponds à ma question sur la sécurité LLM"
├── CHAIN: rag_pipeline 650ms
├── EMBEDDING: query_embed 45ms
├── RETRIEVER: vector_search 120ms
└── 5 chunks récupérés · score max: 0.92
├── RERANKER: cross_encoder 180ms
└── LLM: gpt-4o-mini 285ms · 412 tokens
input_tokens: 320 · output_tokens: 92 · temp: 0.3

Concepts clés

ProjectEspace de travail logique. Regroupe toutes les traces d'une application. Ex : rag-production, chatbot-v2.
TraceSéquence complète d'opérations pour une requête. Identifiée par un trace_id unique. Contient un ou plusieurs spans.
SpanUnité atomique d'une trace. Représente une opération : appel LLM, embedding, retrieval, tool call, guardrail...
SpanKindType du span : LLM, CHAIN, RETRIEVER, EMBEDDING, RERANKER, TOOL, AGENT, GUARDRAIL.
AttributeMétadonnée sur un span : input, output, model, token counts, latence, température, documents récupérés...
DatasetCollection de traces/spans exportés. Sert de base pour les expériences et l'évaluation comparative.
i
OpenTelemetry vs OpenInference
OpenTelemetry est le standard générique (logs, metrics, traces) pour tout système distribué. OpenInference est une convention sémantique au-dessus d'OTel, spécifique aux LLMs : attributs standardisés pour input/output LLM, embeddings, documents, scores de relevance.
Attributs OpenInference - standard
1# Attributs span LLM
2input.value                # Prompt complet envoyé au LLM
3output.value               # Réponse du LLM
4llm.model_name             # "gpt-4o-mini", "claude-3-5-sonnet"
5llm.token_count.prompt     # Tokens input
6llm.token_count.completion # Tokens output
7llm.invocation_parameters  # {"temperature": 0.7, "max_tokens": 512}
8
9# Attributs span RETRIEVER
10retrieval.documents        # Liste JSON des docs récupérés
11# Chaque doc: {"id": "...", "content": "...", "score": 0.92}
12
13# Attributs span EMBEDDING
14embedding.model_name       # "text-embedding-3-small"

instrumentation manuelle - contrôle total
1from opentelemetry import trace
2from openinference.semconv.trace import SpanAttributes, OpenInferenceSpanKindValues
3import json
4
5tracer = trace.get_tracer(__name__)
6
7def rag_pipeline(query: str) -> str:
8    with tracer.start_as_current_span("rag_pipeline") as root:
9        root.set_attribute(SpanAttributes.OPENINFERENCE_SPAN_KIND,
10                           OpenInferenceSpanKindValues.CHAIN.value)
11        root.set_attribute(SpanAttributes.INPUT_VALUE, query)
12
13        with tracer.start_as_current_span("embed_query") as emb:
14            emb.set_attribute(SpanAttributes.OPENINFERENCE_SPAN_KIND,
15                              OpenInferenceSpanKindValues.EMBEDDING.value)
16            vector = embed_text(query)
17
18        with tracer.start_as_current_span("vector_search") as ret:
19            ret.set_attribute(SpanAttributes.OPENINFERENCE_SPAN_KIND,
20                              OpenInferenceSpanKindValues.RETRIEVER.value)
21            docs = retrieve(vector)
22            ret.set_attribute(SpanAttributes.RETRIEVAL_DOCUMENTS, json.dumps(docs))
23
24        with tracer.start_as_current_span("llm_call") as llm:
25            llm.set_attribute(SpanAttributes.LLM_MODEL_NAME, "gpt-4o-mini")
26            answer = generate(query, docs)
27            llm.set_attribute(SpanAttributes.OUTPUT_VALUE, answer)
28
29        root.set_attribute(SpanAttributes.OUTPUT_VALUE, answer)
30        return answer

Évaluations LLM-as-a-Judge

Un LLM juge la qualité des réponses de votre système RAG. Rapide, scalable, et fortement corrélé aux évaluations humaines sur les critères standards.

Faithfulness
La réponse est-elle fondée sur les documents sources ?
Relevance
La réponse répond-elle précisément à la question ?
Toxicity
Contenu offensant ou dangereux détecté dans la réponse ?
Conciseness
La réponse est-elle concise et précise ?

Datasets & Expériences

Construire des datasets depuis les traces de production, puis comparer des configurations via des expériences reproductibles.

Dataset
Collection statique de (input, expected_output) pairs. Golden set pour les expériences.
Experiment
Exécution d'une fonction sur un dataset avec évaluation automatique. Comparaison A/B.

Créer un dataset depuis les traces

python - export traces vers dataset
1import phoenix as px
2
3client   = px.Client()
4spans_df = client.get_spans_dataframe("rag-demo")
5
6# Filtrer les bonnes traces (faithfulness > 0.85)
7good = spans_df[
8    (spans_df["eval.Faithfulness.score"] > 0.85) &
9    (spans_df["span_kind"] == "CHAIN")
10][["input.value", "output.value", "eval.Faithfulness.score"]]
11
12dataset_df = good.rename(columns={
13    "input.value":            "input",
14    "output.value":           "expected_output",
15    "eval.Faithfulness.score": "reference_score",
16})
17
18dataset = client.upload_dataset(
19    dataset_name="golden-set-security",
20    dataframe=dataset_df,
21    input_keys=["input"],
22    output_keys=["expected_output"],
23    metadata_keys=["reference_score"],
24)
